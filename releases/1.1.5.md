## What's New

- some providers have gated channels, they now play — backend runs an invisible reCAPTCHA-warmer in headless chromium, captures the resulting Cloudflare session cookie, and refreshes the IP whitelist every 25 minutes per channel so the AES key endpoint returns real bytes instead of garbage
- Web UI player no longer starts muted on Chrome / Safari (forced unmute + autoplay-rejection fallback so the stream still starts even if the browser blocks unmuted autoplay)
- Streams missing video OR audio at the source automatically fall back to alternate providers — silent feeds no longer leave the player stuck on a video track with no audio
- Custom M3U sources accept "Direct stream (single .m3u8 feed)" as a kind, so pasting a URL adds a single channel pointing at that stream — no more "0 channels detected" on master playlists
- Default Docker image now ships chromium for full gate prevention support

## Docker

```bash
docker pull rebelstream/iptv:1.1.5
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebelstream/iptv:1.1.5
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - ./data:/app/data
    stop_grace_period: 30s
    restart: unless-stopped
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v ./data:/app/data \
  --restart unless-stopped \
  rebelstream/iptv:1.1.5
```

### Updating

```bash
docker pull rebelstream/iptv:1.1.5
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.

## What's New

- Bring-your-own playlists: add custom M3U sources by URL or upload, with optional per-source stream health checking
- Light / dark theme toggle with system-preference detection
- In-dashboard program guide (EPG) viewer, and a new Home landing page
- Redesigned category and league breakdowns on the Dashboard — clickable tiles that deep-link to the filtered Channels / Sports list
- Broadcast-network fallback for ABC / CBS / NBC / FOX — automatic switch to the flagship affiliate when the network's own feed is unavailable, both during scheduled refresh and at stream time
- Streams now fail over to alternate sources automatically when the primary goes offline mid-watch, instead of terminating the session
- Sports events: pregame / live / postgame windows, live score updates, and per-event detail pages with linescores, team colors, and logos
- Show & movie poster artwork enrichment in the EPG
- Stable channel numbering that survives cache purges and restarts
- New Settings toggle to display playlist / EPG URLs using the Docker container hostname, so clients on the same Docker network can be pointed at the stack without hand-editing URLs
- Targeted refresh controls in Settings — refresh just channels, just events, or just the guide
- More reliable after a restart: the guide, channels, and in-flight stream sessions survive container restarts
- Cleaner EPG: focused on the next 24 hours, only tags movies and live sports as categories, and channel names in the guide now match the names in the playlist
- Refresh banner is now readable in light mode
- Faster, lower-latency guide and playlist generation under load
- Copy-to-clipboard URLs are cleaner (single `?key=` param, shared builder across Dashboard and Settings)
- Improved scraper stability and fault tolerance — fetch timeouts, retries, and graceful degradation when an upstream is briefly unreachable

## Docker

```bash
docker pull rebelstream/iptv:1.1.0
```

### Docker Compose

```yaml
services:
    iptv:
        image: rebelstream/iptv:1.1.0
        hostname: iptv
        container_name: iptv
        ports:
            - "8080:8080"
        environment:
            TZ: "America/New_York"
        volumes:
            - ./data:/app/data
        restart: unless-stopped
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v ./data:/app/data \
  --restart unless-stopped \
  rebelstream/iptv:1.1.0
```

### Updating

```bash
docker pull rebelstream/iptv:1.1.0
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.

## What's New

- Custom M3U sources now handle massive playlists (hundreds of thousands of channels) without freezing the dashboard or stalling refresh
- Streams release upstream connections the instant the player disconnects, instead of holding sockets for up to 30 s while prefetched playlists / segments finish downloading
- Finished sports events no longer linger in state after a restart — undatable / past-day events are filtered out at every stage instead of surviving as zombies

## Docker

```bash
docker pull rebelstream/iptv:1.1.3
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebelstream/iptv:1.1.3
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - ./data:/app/data
    stop_grace_period: 30s
    restart: unless-stopped
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v ./data:/app/data \
  --restart unless-stopped \
  rebelstream/iptv:1.1.3
```

### Updating

```bash
docker pull rebelstream/iptv:1.1.3
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
